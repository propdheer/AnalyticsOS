# Guiding Principles

## Engineering Principles

### Architecture Before Implementation

AnalyticsOS should not add code before understanding the design intent.

### Complexity Must Be Earned

A simple design is preferred until there is a real reason to add complexity.

### Documentation Equals Code

A feature is not complete until it is documented.

### Every Component Must Justify Its Existence

If a component can be removed without reducing product value, it should not exist.

### Core Must Not Depend on Plugins

Plugins can depend on core contracts. Core must remain generic.

## AI Principles

### Human-in-the-Loop

AI suggests. Humans approve.

### Explainability

Every important answer should eventually show source context, memory context, and confidence.

### Local-First by Default

Cloud AI can be optional, but local operation must remain a first-class path.

## Product Principles

### Knowledge Compounds

AnalyticsOS should become more valuable each week it is used.

### Projects, Not Just Documents

The system should understand real professional objects such as projects, datasets, business rules, dashboards, and decisions.

### Search Before Chat

Excellent retrieval is more important than a clever chatbot.
