# v0.5.0-alpha Durable Platform

## Purpose

This release hardens AnalyticsOS for daily use across two machines while keeping the existing FastAPI + Next.js + local-first architecture.

## Important Design Decision

AnalyticsOS does **not** force one runtime location.

Each machine can choose its own runtime path:

| Machine | Recommended Runtime Location |
|---|---|
| Office laptop | OneDrive runtime folder |
| Personal laptop | `%LOCALAPPDATA%\AnalyticsOS` |
| Any machine | Any custom folder you choose |

## Configure Runtime Paths

Run:

```powershell
.\scripts\configure-runtime-paths.ps1
```

Choose:

```text
1. Local only
2. OneDrive
3. Custom paths
```

This writes the selected paths to `.env`:

```text
ANALYTICSOS_DATA_DIR=...
ANALYTICSOS_KNOWLEDGE_DIR=...
```

Restart the backend after changing these values.

## Optional Migration

To copy existing repo `data/` and `knowledge/` into a chosen target:

```powershell
.\scripts\migrate-runtime-data.ps1 -TargetDataDir "C:\Path\data" -TargetKnowledgeDir "C:\Path\knowledge"
```

For personal local-only migration:

```powershell
.\scripts\migrate-runtime-to-localappdata.ps1
```

## Snapshot Sync

Push current state to a snapshot folder:

```powershell
.\scripts\sync-push.ps1
```

Pull a snapshot:

```powershell
.\scripts\sync-pull.ps1
```

`sync-pull.ps1` asks for confirmation before importing.

## Settings

Use the Settings screen to configure:

- Data directory
- Knowledge directory
- Sync directory
- Ollama URL and model
- AnythingLLM URL, API key, workspace slug
- Obsidian vault path
