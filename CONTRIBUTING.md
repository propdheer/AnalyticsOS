# Contributing to AnalyticsOS

AnalyticsOS is designed as an open-source-quality engineering project from day one.

## Engineering Standard

Every contribution should be:

- Documented
- Tested
- Typed where applicable
- Modular
- Easy to review
- Aligned with the project architecture

## Commit Style

Use Conventional Commits:

```text
feat(repository): initialize foundation
docs(handbook): add manifesto
fix(api): handle health check error
test(backend): add health endpoint test
chore(ci): configure workflow
```

## Pull Request Checklist

Before merging:

- [ ] Code runs locally
- [ ] Tests pass
- [ ] Documentation updated
- [ ] Changelog updated
- [ ] ADR added if architecture changed
- [ ] No secrets or private data committed
