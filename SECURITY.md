# Security Policy

AnalyticsOS is local-first and privacy-first.

## Security Principles

- No private files should be committed to Git.
- Secrets must live in `.env` files and remain untracked.
- Local services should bind to localhost unless explicitly configured otherwise.
- Browser capture features must be consent-based.
- Automation features must be human-approved by default.

## Reporting Security Issues

Until a public process is established, report issues privately to the repository owner.
